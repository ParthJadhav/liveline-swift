# Product

## Register

brand

## Users

iOS and SwiftUI developers evaluating a charting package for real-time financial, telemetry, or product-data interfaces. They are scanning the README to decide whether the renderer looks native, capable, and safe to adopt.

## Product Purpose

Liveline Swift provides native SwiftUI real-time charts with line, candlestick, and multi-series modes, smooth interpolation, scrubbing, live badges, time windows, and market-style overlays. The README should make the library's rendering quality obvious before a developer reads the API.

## Brand Personality

Precise, native, restrained. The visual language should feel like a serious Swift package with tasteful product craft, not a trading app advertisement.

## Anti-references

Avoid crypto-dashboard theatrics, neon grid backgrounds, overlapping fake glass cards, loud gradient posters, excessive pill badges, and Markdown table galleries that make screenshots feel trapped in a spreadsheet.

## Design Principles

- Show the real renderer first; screenshots are proof, not decoration.
- Let SwiftUI-native polish lead: quiet surfaces, exact spacing, legible type, and restrained color.
- Make the examples easy to compare without surrounding them with UI clutter.
- Keep marketing imagery reproducible from deterministic Storybook captures.
- Prefer calm confidence over visual noise.

## Accessibility & Inclusion

README media should remain legible on light and dark GitHub themes, include useful alt text, avoid relying on color alone for meaning, and keep motion out of static documentation assets.
